# Lawful Visual Quality Rule

## Purpose

Every visible geometry image on Galway Geometry must preserve the lawfulness of the field.

The site may be beautiful, dense, colourful, public, and playful, but it must not display geometry that contradicts the substrate.

This is a quality-control rule, not a front-page manifesto.

## Core Rule

No image should betray the lawful field.

A page may use HTML and SVG freely, but visual geometry should be generated from, or visibly consistent with, the lawful substrate.

The lawful substrate is the unit triangular field generated by a fixed radius.

Use the native coordinate form:

```text
P(q,r) = q*u + r*v
u = (1,0)
v = (1/2, sqrt(3)/2)
```

For screen projection, use:

```text
x = origin_x + unit * (q + r/2)
y = origin_y - unit * (r * sqrt(3)/2)
```

All nearest-neighbour edges in the substrate have length one unit.

## Allowed Visual Geometry

Preferred geometry includes:

- triangular lattice lines;
- hexagonal station sets from the first circle;
- equal-radius circles centred on lawful lattice stations;
- unit chords between lawful neighbouring stations;
- diameters through lawful opposite stations;
- equilateral cells whose vertices are lawful lattice points;
- field drawings composed from lawful cells;
- mandalas derived from lawful sixfold structure;
- tessellation sheets built from the field;
- plate events that compose and decompose lawful relations.

## Warning Signs

A visual should be revised if it contains:

- arbitrary decorative circles not centred on lawful stations;
- arbitrary polygons whose vertices are not on the lawful field;
- pretty symmetry that is not generated by the fixed-radius structure;
- poster geometry that only resembles the substrate;
- preview images whose coordinates are hand-placed without field logic;
- rectangles used as geometric claims rather than UI panels or document frames.

UI panels, cards, buttons, and page layout may use ordinary HTML rectangles. The rule governs geometry images and visual claims, not interface boxes.

## Page-Level Test

Before a page is considered visually acceptable, ask:

1. Is every geometry preview built from lawful coordinates, lawful circles, lawful cells, or lawful station relations?
2. Are all displayed circles using a radius that belongs to the construction being shown?
3. Are all centres and vertices explainable as lattice points, circle centres, boundary stations, or UI-only elements?
4. Is any decoration being mistaken for generated geometry?
5. Would this preview still make sense if inspected as part of the field?

If the answer fails, the visual should be corrected.

## Implementation Rule

Future pages should prefer generated inline SVG based on the triangular lattice projection.

Do not use generic stock images.

Do not use decorative geometry that merely looks plausible.

The site should not announce this rule loudly to visitors. It should simply obey it.
